# 设计文档

## 游戏设计理念

### 核心概念
"破局"——在绝境中寻找生路。不是考验历史知识，而是考验在信息不完整、时间有限、对手在暗处的情境下的决策能力。

### 设计原则
1. **历史真实感**：基于真实历史事件，但主角可虚构
2. **信息不完整**：玩家只能看到局部信息，像真实历史中一样
3. **多重结局**：没有"正确"答案，只有"不同后果"
4. **失败友好**：失败后可直接重试，不为难玩家
5. **顿悟时刻**：设计"哦！原来如此！"的破局点

## 视觉风格

### 色彩系统
- **主色**：#2C2826（墨色）- 沉稳、历史感
- **背景**：#F4EFE6（和纸奶白）- 古籍质感
- **强调色**：#C03C28（朱砂红）- 印章、警示
- **金色**：#B8956A（碑刻金）- 点缀、高级感

### 字体
- **标题**：Noto Serif SC（宋体风格）
- **正文**：Noto Sans SC（黑体风格）
- **备用**：PingFang SC, Microsoft YaHei, SimSun

### 交互风格
- **卡片式**：每步一个固定大小卡片
- **滑动动画**：卡片切换有流畅过渡
- **步骤指示**：底部圆点显示进度

## 关卡设计规范

### 结构
1. **起始场景**：介绍背景、人物、困局
2. **选择分支**：2-3个选项，每个引向不同路径
3. **中间场景**：展开剧情，提供新信息
4. **结局场景**：展示结果，附历史注脚

### 困局类型
- 被借刀杀人
- 骑墙困境
- 信息差陷阱
- 时间压力
- 道德两难

### 破局思路设计
- 必须有至少一条"破局"路径
- 破局点通常是反直觉的
- 常见破局策略：
  - 让对方的"说法"失效
  - 制造竞价局面
  - 引入第三方力量
  - 改变问题定义

## 技术架构

### 前端
- 纯HTML5 + CSS3 + JavaScript
- 无框架依赖
- 单文件部署

### 数据结构
```javascript
{
  id: '关卡唯一标识',
  era: '朝代 · 年份',
  name: '关卡名称',
  desc: '简短描述',
  tag: '困局类型标签',
  scenes: [
    {
      id: '场景标识',
      text: 'HTML格式的故事文本',
      choices: [
        { text: '选项文字', next: '下一个场景ID', tag: '选项标签' }
      ],
      result: '结局类型', // win/win_alt/neutral/lose
      historyNote: '历史注脚'
    }
  ]
}
```

## 性能优化

- 字体本地化（减少外部请求）
- 图片懒加载
- CSS动画使用transform（GPU加速）
- 避免强制同步布局

## 可访问性

- 支持屏幕阅读器
- 键盘导航
- 高对比度模式
- 字体大小调整
